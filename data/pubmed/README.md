# PubMed Data Directory — Wikimedica

This directory contains processed PubMed surveillance records generated by the daily automated search pipeline (`scripts/pubmed/pubmed-daily-search.py`).

---

## Directory Contents

```
data/pubmed/
├── README.md                   ← this file
├── YYYY-MM-DD.jsonl            ← daily output files (one per run date)
├── pending-review.jsonl        ← MEDIUM relevance records awaiting editorial review
├── processed_pmids.txt         ← flat list of all previously processed PMIDs
└── error.log                   ← pipeline error log
```

---

## Record Schema

Each line in a `.jsonl` file is a valid JSON object with the following fields:

| Field | Type | Required | Description |
|---|---|---|---|
| `pmid` | string | ✅ | PubMed Identifier |
| `title` | string | ✅ | Article title |
| `abstract` | string | ✅ | Full abstract text (may be empty string if no abstract) |
| `authors` | array of strings | ✅ | Author names in "Lastname I" format |
| `journal` | string | ✅ | Journal name (NLM title abbreviation) |
| `year` | integer | ✅ | Publication year |
| `publication_date` | string | ✅ | Full publication date (ISO 8601 where available) |
| `publication_types` | array of strings | ✅ | PubMed publication type tags (e.g., "Randomized Controlled Trial") |
| `mesh_terms` | array of strings | ✅ | MeSH descriptor terms from PubMed indexing |
| `specialty_tags` | array of strings | ✅ | Wikimedica specialty labels (mapped from MeSH terms and keywords) |
| `relevance_score` | integer | ✅ | Calculated relevance score (0–100) |
| `relevance_class` | string | ✅ | Classification: `HIGH`, `MEDIUM`, or `LOW` |
| `action_taken` | string | ✅ | One of: `issue_opened`, `logged_for_review`, `stored_only` |
| `github_issue_url` | string or null | ✅ | URL of the created GitHub Issue (if applicable) |
| `processed_at` | string | ✅ | ISO 8601 timestamp of when the record was processed |
| `source` | string | ✅ | Always `"pubmed"` |
| `search_query_matched` | string | ✅ | The search query that matched this record |
| `language` | string | ❌ | Article language code (ISO 639-1) |
| `doi` | string or null | ❌ | Digital Object Identifier |
| `pubmed_url` | string | ✅ | Direct link: `https://pubmed.ncbi.nlm.nih.gov/{pmid}/` |

### Example Record

```json
{
  "pmid": "38000001",
  "title": "Sex differences in acute myocardial infarction presentation and outcomes: a systematic review",
  "abstract": "Background: Women presenting with acute myocardial infarction (AMI)...",
  "authors": ["Mueller A", "Schmidt B", "Weber C"],
  "journal": "Eur Heart J",
  "year": 2024,
  "publication_date": "2024-11-15",
  "publication_types": ["Systematic Review", "Meta-Analysis"],
  "mesh_terms": ["Myocardial Infarction", "Sex Factors", "Women"],
  "specialty_tags": ["Kardiologie", "Gender-Medizin"],
  "relevance_score": 82,
  "relevance_class": "HIGH",
  "action_taken": "issue_opened",
  "github_issue_url": "https://github.com/atmedical/DIG-wikimedica/issues/42",
  "processed_at": "2024-11-16T06:15:33Z",
  "source": "pubmed",
  "search_query_matched": "Kardiologie_core",
  "language": "en",
  "doi": "10.1093/eurheartj/ehad000",
  "pubmed_url": "https://pubmed.ncbi.nlm.nih.gov/38000001/"
}
```

## processed_pmids.txt

A plain-text file containing one PMID per line. The pipeline checks this file before processing to avoid reprocessing already-seen records. This file is appended to on each run and committed to the repository.

---

## Retention Policy

- Daily `.jsonl` files are retained **indefinitely** in the repository (they serve as an audit trail).
- If storage becomes a concern, files older than **2 years** may be moved to an archived branch or external storage.
- `pending-review.jsonl` is cleared (records removed) when an editorial decision has been recorded for each entry.
- `error.log` is rotated monthly; old logs are archived.

---

## Linkage to Content Articles

When a PubMed record leads to a content update or new article, the PMID is added to the article's YAML frontmatter under `pubmed_ids`. The GitHub Issue created for the record is referenced in the PR description. This creates a traceable chain:

```
PubMed record → GitHub Issue → Pull Request → Article frontmatter (pubmed_ids)
```
